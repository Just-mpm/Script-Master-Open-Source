## Alto

### [A-01] Erro 500 INTERNAL do TTS não é retentado pelo `withRetry`

**Arquivo:** `src/lib/rate-limiter.ts:37`  
**Confidence:** 95

`RETRYABLE_STATUS_CODES = new Set([429, 503, 504])` — não inclui `500`. Gemini TTS pode retornar 500 intermitentemente (confirmado via NotebookLM Gemini API docs). Quando ocorre, a geração falha sem retry e o usuário perde todo o progresso dos chunks anteriores.

**Fix:** Adicionar `500` ao `RETRYABLE_STATUS_CODES` (1 linha).

### [I-01] Chat sessions >900KB retornam versão desatualizada do Firestore

**Arquivos:** `src/lib/db/chats.ts:44-84`, `src/features/assistant/Assistant.tsx:101`  
**Confidence:** 90

`saveChatSession` faz fallback para IndexedDB quando doc >900KB, mas **não remove doc antigo do Firestore**. `getChatSessions(userId)` consulta apenas Firestore, retornando versão parcial. Cenário reproduzível de **perda de dados permanente**: chat cresce → migra para IndexedDB → refresh → histórico mostra versão antiga → novo save sobrescreve Firestore sem mensagens recentes.

**Fix:** `getChatSessions(userId)` deve buscar Firestore E IndexedDB, deduplicar por `session.id` preferindo `updatedAt` mais recente. Ou deletar doc Firestore quando migrar para IndexedDB.

### [I-03] Auto-save do chat sem fallback a IndexedDB em erro Firestore

**Arquivos:** `src/lib/db/chats.ts:57-67`, `src/hooks/useAssistant.ts:118`  
**Confidence:** 82
