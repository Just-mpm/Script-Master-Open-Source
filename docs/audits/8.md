## WARNING (P1)

### [P1] AnimationControls — `setTimeout` sem cleanup no auto-start de gravação batch

**Arquivo:** `src/features/speed-paint/components/canvas/AnimationControls.tsx:237-247`
**Confidence:** 92/100
**Impacto:** Gravação inesperada de vídeo (~12Mbps stream) se o usuário navegar para outra rota durante o delay de 800ms no modo batch record

**Problema:**
O `useEffect` que dispara gravação automática usa `setTimeout(() => {...}, 800)` sem cleanup function. Se o componente desmontar durante o delay (navegação rápida), a callback executa com `handleDownloadVideoRef.current()` que acessa o DOM. O guard `useAnimationStore.getState().batchMode === 'record'` não protege contra desmontagem — a store persiste entre montagens. Ao remontar, `isRecording` é `false`, fazendo a condição `!isRecording` passar novamente, podendo causar execução dupla.

Nota: outros 2 `setTimeout` no mesmo arquivo já possuem cleanup — este é um oversight.

**Correção sugerida:** Adicionar `return () => clearTimeout(timer)` no useEffect:
```tsx
const timer = setTimeout(() => { ... }, 800);
return () => clearTimeout(timer);
```

## SUGGESTION (P2)

### [P2] Library.tsx — blob URL de download de áudio não revogada após uso

**Arquivo:** `src/components/Library.tsx:592-598`
**Confidence:** 82/100
**Impacto:** ~5-10MB por blob URL não revogado (WAV 24kHz). Acumulação de ~50-100MB após 10 downloads até o desmonte do componente

**Problema:**
O handler de download de áudio cria um blob URL e adiciona ao `blobUrlsRef` para cleanup futuro, mas nunca revoga após o download ser concluído. Cleanup acontece apenas no desmonte do componente ou ao trocar projeto expandido. Áudios WAV gerados por TTS podem ter ~5-10MB cada.

**Correção sugerida:** Revogar o blob URL após `downloadFile` resolver:
```tsx
void downloadFile(url, name).then(() => {
  if (isBlob) { URL.revokeObjectURL(url); blobUrlsRef.current.delete(url); }
});
```

| AnimationPlayer — re-render 20x/s | **Premissa incorreta.** `progress` não está nos seletores do componente (usa `getState()` no rAF loop, bypassando React). Componente só re-renderiza em eventos infrequentes. |
